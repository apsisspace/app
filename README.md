# Apsis Space

> *An AI-native browser-based satellite tracker. See every active satellite in orbit. Ask questions. Understand space.*

![Apsis Space](./docs/screenshot.png)

## Features

- Real-time 3D visualization of ~10,000 active satellites
- AI chat assistant for natural-language queries (powered by Claude Haiku)
- Color-coded by orbital inclination
- Search and filter by satellite name or NORAD ID
- Detailed side panel with orbital parameters
- Clean, opinionated UI with minimal Earth mode

## Tech Stack

- Frontend: Vite, React 18, TypeScript, CesiumJS (via resium)
- Propagation: satellite.js (SGP4)
- State: Zustand, TanStack Query
- Styling: Tailwind CSS
- AI: Anthropic Claude Haiku via Vercel Edge Functions
- Rate limiting: Upstash Redis
- Hosting: Vercel

## Quick Start

1. **Clone the repository:**
   ```bash
   git clone https://github.com/yourusername/apsis-space.git
   cd apsis-space
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Configure environment variables:**
   Copy the example environment file and fill in your keys (see below):
   ```bash
   cp .env.example .env.local
   ```

4. **Run the development server:**
   ```bash
   npm run dev
   ```
   Open `http://localhost:5173` in your browser.

## Environment Variables

The application requires specific environment variables for imagery and server-side features. Set these in your `.env.local` file:

| Variable | Purpose | Where to get it |
| :--- | :--- | :--- |
| `VITE_CESIUM_ION_TOKEN` | Enables premium satellite imagery and terrain rendering in Cesium. Without it, Apsis Space falls back to OpenStreetMap imagery. | Free token at [ion.cesium.com](https://ion.cesium.com/) |
| `ANTHROPIC_API_KEY` | Server-side only. Used by the `/api/query` AI layer to power the Ask Apsis chat assistant (Claude Haiku). | [console.anthropic.com](https://console.anthropic.com/) |
| `UPSTASH_REDIS_REST_URL` | Server-side only. Used by `/api/query` for per-IP rate limiting and global budget tracking. | Free DB at [upstash.com](https://upstash.com/) |
| `UPSTASH_REDIS_REST_TOKEN`| Server-side only. Used by `/api/query` to authenticate with Redis. | Free DB at [upstash.com](https://upstash.com/) |

*(Note: Variables without the `VITE_` prefix are only exposed to the Vercel Edge Functions, not the browser.)*

## Architecture

Apsis Space minimizes load on external providers through aggressive caching and intelligent client-side rendering. When the browser loads, it checks a `localStorage` cache for the Celestrak satellite catalog. On a cache miss, it hits a Vercel Edge Function (`/api/catalog`) which fetches the data from Celestrak and caches it. The client then passes this data to `satellite.js` to compute SGP4 propagations. CesiumJS handles the 3D rendering of the Earth and the calculated satellite positions.

The AI chat feature flows similarly but in real time. User input from the chat panel is sent to `/api/query`. This Edge Function checks both a per-IP rate limit and a global daily budget stored in Upstash Redis. If the limits are not exceeded, the query is forwarded to Anthropic's Claude API. The response is streamed back to the user, and the total token cost is tallied in Redis.

## Roadmap

- **Done:**
  - Real-time 3D Earth rendering with client-side SGP4 propagation.
  - Active catalog integration (loading ~10,000 satellites smoothly).
  - Click-to-inspect UI with real-time altitude, velocity, and orbital parameters.

  - "Ask Apsis" AI chat layer with contextual awareness of selected satellites.
  - Server-side rate limiting and abuse controls for AI queries.

  - User accounts and saved custom views.
  - Higher query limits for authenticated/pro users.
  - Premium imagery and terrain via Cesium ion integration for Pro tier.

  - Live visibility predictions (requires user location/time capabilities).
